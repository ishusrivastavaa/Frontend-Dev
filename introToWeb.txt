2. Simple Diagram: Client-Server Model
+-------------+           Request (HTTP)           +-----------------+
|             | ---------------------------------> |                 |
|   Browser   |                                    |    Web Server   |
|  (Client)   | <--------------------------------- |                 |
+-------------+           Response (HTML)          +-----------------+


Explanation:

The client (browser) sends a request (like typing www.google.com).

The server processes the request and sends back the page content (HTML/CSS/JS).



3. How a Browser Requests and Displays a Web Page

Step-by-Step Process:

You type a URL (e.g., www.example.com) in the browser.

The browser contacts the DNS server to find the IP address of that domain.

It sends an HTTP request to the web server at that IP address.

The web server processes the request and sends back an HTTP response containing the HTML.

The browser parses the HTML, downloads CSS, JS, and images.

The rendering engine builds and displays the final web page visually.

5. What is a Web Server?

A web server is software or hardware that serves web pages to users upon request.
It receives HTTP requests and sends back HTTP responses.

Examples:

Apache HTTP Server

Nginx

Microsoft IIS

Node.js (using Express)

LiteSpeed

8. Difference between Static and Dynamic Websites
Type	Description	Example
Static Website	Same content for every visitor; built with only HTML, CSS, JS.	A portfolio website with fixed pages.
Dynamic Website	Content changes based on user interaction or data. Uses backend + database.	Facebook, YouTube, or an e-commerce site showing personalized results.
